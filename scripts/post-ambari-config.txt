Ambari Setup (steps required after deployment)
==============================================

Ambari Dashboard 
================
# Increase ResourceManager heap size to 3 MB (if any alerts exist)

Master Machine
==============
# Retrieve Mysql temporary password
sudo grep 'temporary password' /var/log/mysqld.log

# Login to Mysql using the password retrieved above and enter the following commands
ALTER USER 'root'@'localhost' IDENTIFIED BY 'Xxxxxx@xxxx9';

CREATE USER 'hive'@'x.x.x.x' IDENTIFIED BY 'Xxxxxx@xxxx9';
GRANT ALL PRIVILEGES ON *.* TO 'hive'@'x.x.x.x' WITH GRANT OPTION;
CREATE USER 'hive'@'%' IDENTIFIED BY 'Xxxxxx@xxxx9';
GRANT ALL PRIVILEGES ON *.* TO 'hive'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;


Slave Machines
==============
# If there is a 'hadoop' directory under /usr/hdp in slave nodes, remove them.
rm -fr /usr/hdp/hadoop/

Flume Machines
==============
# One possible error handling in case of existing alerts after setting up Flume:
mkdir /var/run/flume
chown flume:hadoop /var/run/flume
